Per the Single-Plane-Pull architecture the prop must live INSIDE its host chamber zone (HolographicChamber's position: relative container), not the viewport. Fix removed the !important override + removed the React portal to document.body. The prop now renders inline in the React tree with position: absolute, resolving its coordinates against the chamber container. It can never paint outside the chamber, can never overlap unrelated content. No new layer, no overlay added. Pure stacking-context correction.
